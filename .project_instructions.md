# Sistema Distribuidora - Fase 1 (MVP)

## Descripción del Proyecto
Sistema de gestión completo para distribuidora con autenticación de múltiples usuarios, control de inventario con transacciones atómicas, registro de ventas, reportes y personalización de colores. Configurado como PWA para instalación en dispositivos Android.

## Tecnologías Utilizadas
- **Framework**: Next.js 14 con App Router
- **Base de Datos**: PostgreSQL con Prisma ORM
- **Autenticación**: NextAuth.js v4 con strategy JWT
- **Estilos**: Tailwind CSS
- **Gráficos**: Recharts
- **PWA**: Manifest.json + Service Worker

## Arquitectura de la Base de Datos

### Modelos Principales:
1. **User** - Usuarios del sistema (Admin/Empleado)
2. **Product** - Catálogo de productos con SKU único
3. **Category** - Categorías de productos
4. **Sale** - Ventas realizadas
5. **SaleItem** - Items individuales de cada venta
6. **InventoryMovement** - Historial de movimientos de inventario
7. **AppSettings** - Configuración de colores personalizables

### Transacciones Críticas:
- **Ajustes de inventario** (`/api/inventario/ajustar`): Usa `prisma.$transaction()` para garantizar atomicidad
- **Registro de ventas** (`/api/ventas`): Transacción que valida stock, crea venta, actualiza inventario y registra movimientos

## Estructura de Roles y Permisos

### Administrador (ADMIN):
- Acceso completo a todas las funcionalidades
- Gestión de productos (CRUD)
- Control de inventario (ajustes manuales)
- Reportes de ventas
- Gestión de usuarios
- Personalización de colores
- Registro de ventas

### Empleado (EMPLEADO):
- Vista de productos (solo lectura)
- Registro de ventas
- Dashboard con métricas básicas

## Funcionalidades Implementadas

### 1. Autenticación
- Login con email y password
- Roles: ADMIN y EMPLEADO
- Sesiones con JWT
- Middleware para protección de rutas

### 2. Dashboard
- Métricas: ventas del día, ventas del mes, total productos, stock bajo
- Gráficos de ventas recientes y productos más vendidos
- Alertas visuales de stock bajo
- Animaciones de contadores

### 3. Gestión de Productos
- CRUD completo (solo Admin)
- Búsqueda por nombre o SKU
- Filtrado por categoría
- Campos: SKU, nombre, descripción, precio, stock, stock mínimo, categoría
- Validación de SKU único

### 4. Control de Inventario (Solo Admin)
- Visualización de stock actual
- Tres tipos de movimientos:
  - **ENTRADA**: Incrementa stock
  - **SALIDA**: Decrementa stock con validación
  - **AJUSTE**: Establece stock a valor específico
- Historial completo de movimientos
- Registro de razón y usuario que realiza el movimiento

### 5. Registro de Ventas
- Formulario multi-producto
- Validación de stock disponible en tiempo real
- Cálculo automático de totales
- Descuento automático de inventario al completar venta
- Registro de movimiento por cada producto vendido
- Lista de ventas recientes en sidebar

### 6. Reportes de Ventas (Solo Admin)
- Filtros por período: Hoy, Esta Semana, Este Mes, Rango Personalizado
- Agrupación por empleado
- Detalles de cada venta con productos
- Exportación a CSV
- Métricas generales: total vendido, cantidad de ventas

### 7. Gestión de Usuarios (Solo Admin)
- Crear nuevos usuarios (Admin o Empleado)
- Editar información de usuarios
- Activar/Desactivar usuarios
- Cambio de contraseña
- Visualización de rol y estado

### 8. Configuración (Solo Admin)
- Personalización de 3 colores:
  - Color Primario
  - Color Secundario
  - Color de Acento
- Selector visual de colores
- Aplicación inmediata de cambios
- Persistencia en base de datos

### 9. PWA
- Manifest.json configurado
- Service Worker para caché básico
- Iconos SVG en 8 tamaños (72 a 512px)
- Instalable en Android
- Funcionalidad offline básica

## Características Técnicas Importantes

### Prevención de Race Conditions:
- Uso de transacciones de Prisma para operaciones críticas
- Lock implícito al leer productos dentro de transacción
- Validación de stock antes de cada venta

### Optimizaciones:
- Índices en campos frecuentemente consultados (email, sku, categoryId, saleDate)
- Queries optimizadas con includes selectivos
- Server Components por defecto, Client Components solo donde se necesita interactividad

### Seguridad:
- Passwords hasheados con bcrypt
- Validación de roles en APIs
- Middleware de NextAuth para protección de rutas
- Validación de datos en formularios

## Rutas de la Aplicación

### Públicas:
- `/login` - Página de inicio de sesión

### Protegidas (requieren autenticación):
- `/dashboard` - Panel principal
- `/productos` - Lista de productos (todos)
- `/productos/nuevo` - Crear producto (solo Admin)
- `/productos/[id]/editar` - Editar producto (solo Admin)
- `/inventario` - Control de inventario (solo Admin)
- `/ventas` - Registro de ventas (todos)
- `/reportes` - Reportes de ventas (solo Admin)
- `/usuarios` - Gestión de usuarios (solo Admin)
- `/configuracion` - Configuración de colores (solo Admin)

## API Endpoints

### Autenticación:
- `POST /api/auth/[...nextauth]` - NextAuth handlers
- `POST /api/signup` - Registro de usuarios

### Productos:
- `GET /api/productos` - Listar productos
- `POST /api/productos` - Crear producto
- `GET /api/productos/[id]` - Obtener producto
- `PUT /api/productos/[id]` - Actualizar producto
- `DELETE /api/productos/[id]` - Eliminar producto

### Inventario:
- `POST /api/inventario/ajustar` - Ajustar inventario (con transacción)

### Ventas:
- `POST /api/ventas` - Registrar venta (con transacción)

### Reportes:
- `GET /api/reportes/ventas` - Obtener reportes con filtros

### Usuarios:
- `POST /api/usuarios` - Crear usuario
- `PUT /api/usuarios/[id]` - Actualizar usuario
- `PATCH /api/usuarios/[id]` - Cambiar estado activo

### Configuración:
- `POST /api/configuracion/colores` - Actualizar colores

## Notas de Desarrollo
- El schema de Prisma usa enums para roles (ADMIN, EMPLEADO) y tipos de movimiento (ENTRADA, SALIDA, AJUSTE)
- No se usan migraciones de Prisma, solo `prisma db push` para sincronizar schema
- El seed script (`scripts/seed.ts`) usa upsert para evitar duplicados
- Los iconos PWA son SVGs generados automáticamente con script Node.js
- Service Worker implementa estrategia cache-first para rutas principales
